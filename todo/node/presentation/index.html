REST APIs

There are several REST APIs that need to be implemented in order to make our ToDo application work.

  • GET /api/todos - This API should return a JSON array of all ToDo objects
  • POST /api/todos - This API can be used to create a new ToDo. The POST body should contain the JSON representation of a ToDo. The API should return the same JSON ToDo object but with a unique id to identify it.
  • PUT /api/todos/{id} - This API can be used to update an existing ToDo. The id in the path represents the id of the ToDo you would like to update. The PUT body should include a JSON representation of the ToDo with the updated values.
  • DELETE /api/todos/{id} - This API can be used to delete a ToDo. The id in the path represents the id of the ToDo to delete.

What About Persistence

We are going to want to persist our ToDos somewhere so when a user comes back to the application or refreshes the page they will see the same data. In our application we will choose to use Cloudant.

  1. From the application dashboard in Bluemix select the Add A Service button. This will bring you back to the Bluemix catalog.
  2. In the data management section of the catalog select the Cloudant service.
  3. In the service dialog give the service the name "todo-cloudant" and click create.
  4. You will be asked to restage the application click cancel for now.
  5. You should now see a tile for Cloudant in your application dashboard, click the arrow next to the word Credentials. There are the credentials for your application to access Cloudant.
